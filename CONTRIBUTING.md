# Contributing to awesome-glm-5.2-usecases

Thank you for helping improve this GLM-5.2 usecase repository.

## What We Accept

We accept public, source-linked GLM-5.2 cases with concrete evidence:

- benchmark results with method or leaderboard context
- coding-agent workflows with task details
- hands-on demos with visible output
- provider or tool integrations with a public launch post
- pricing, cost, deployment, or quota observations
- limitations, safety, latency, or caveat reports

## Required Case Metadata

Every proposed case must include:

- public source URL
- creator handle and profile URL
- concise title
- usage takeaway
- source-grounded notes
- type: Demo, Tutorial, Evaluation, Integration, Benchmark, or Limit
- publication date in `YYYY-MM-DD`

## Repository Policy

Raw source exports and internal curation datasets are not stored in this public repository. Public traceability is carried by the source links in each case heading.

## Formatting

Use the existing README format:

```md
<a id="case-x"></a>
### Case X: [Title](source-link) (by [@author](author-link))

**Usage takeaway.**

Source-grounded notes.

Type: Demo | Date: YYYY-MM-DD
```

Do not invent prompts, benchmark scores, or unsupported claims.
